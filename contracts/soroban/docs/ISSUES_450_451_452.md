# Contract: Pausable Data Submission (#450)

Global submission pause stops all mutating data entrypoints while read-only queries continue to work.

## Functions

| Function | Access | Description |
|----------|--------|-------------|
| `pause_submissions(caller, reason)` | Admin | Pause all data submissions |
| `resume_submissions(caller)` | Admin | Resume submissions |
| `is_submissions_paused()` | Public read | Returns pause flag |
| `get_submission_pause_state()` | Public read | Full pause state |
| `get_submission_pause_history()` | Public read | Pause/unpause audit trail |

## Guarded entrypoints

When paused, the following panic with `data submissions are globally paused`:

- `submit_health`, `submit_health_batch`, `submit_calculated_health`, `submit_health_multi_source`
- `submit_price`
- `record_supply_mismatch`, `record_liquidity_depth`, `record_pool_state`

Reads such as `get_price`, `get_asset_status_rollup`, and metadata queries are unaffected.

## Events

- `sub_pau` — emitted when submissions are paused
- `sub_res` — emitted when submissions are resumed

# Contract: Event Query (#452)

`query_contract_events(filter)` returns recent contract events in a stable shape.

## Filter fields

| Field | Description |
|-------|-------------|
| `event_type` | Optional exact event type match |
| `asset_code` | Optional subject/asset code match |
| `from_ordering_key` | Cursor; use `0` from the beginning |
| `limit` | Max results per call (default 50, max 100) |

Results are ordered by ascending `ordering_key`. Each record includes `schema_version` for deterministic decoding.

Backward-compatible helpers remain available: `get_replay_events`, `get_replay_log_size`, `get_replay_schema_version`.

# Contract: Asset Metadata Update (#451)

`update_asset_metadata` edits metadata for a registered asset without recreating the registration entry.

## Rules

- Admin authorization required
- Asset must already be registered (`AssetHealth` entry exists)
- Field length limits enforced (`name`, `symbol`, `description`, `url`, `change_reason`)
- Each update increments `version` and appends to metadata history
- Emits `meta_up` event with `(version, timestamp)`

Query helpers: `get_asset_metadata`, `get_asset_metadata_history`.
